# exam2 실행 순서

## 구성
- `exam2.ino` : 아두이노 메인 스케치. `Motor` 객체 2개(motorL, motorR)를 생성해 시리얼 명령으로 제어.
- `Motor.h` / `Motor.cpp` : L298N 모터 1채널 제어 클래스 (PWM 없이 방향핀 2개(DIR1/DIR2)로 정방향/역방향/정지만 on/off 제어).
- `motor_ui.py` : 파이썬 tkinter UI. 전진/후진/좌/우 버튼으로 모터 제어.

## 1. 하드웨어 연결
1. L298N 모터 드라이버에 모터 2개 연결 (왼쪽/오른쪽).
2. L298N의 모터 전원 입력에는 18650 배터리를 연결.
3. 아두이노 로직 전원은 배터리와 분리된 별도 전원(USB 등) 사용, GND는 배터리·L298N·아두이노 모두 공통으로 연결.
4. 왼쪽 모터: IN1 → 아두이노 5번 핀, IN2 → 6번 핀.
5. 오른쪽 모터: IN3 → 아두이노 10번 핀, IN4 → 11번 핀.
6. ENA/ENB(속도 인에이블 핀)는 사용하지 않으므로 L298N 보드에서 항상 HIGH가 되도록 고정 배선(점퍼 캡 장착 등) — PWM 속도 제어 없이 항상 최대 속도로 켜짐.
   - 실제 배선이 다르면 `exam2.ino`의 `MOTOR_L_DIR1/DIR2`, `MOTOR_R_DIR1/DIR2` 값을 맞게 수정.

## 2. 아두이노 업로드
1. 아두이노 IDE에서 `exam2.ino` 열기 (같은 폴더의 `Motor.h`, `Motor.cpp`가 탭으로 함께 열려야 함).
2. 보드/포트 선택 후 업로드.
3. 업로드 완료 후 시리얼 모니터는 반드시 닫는다 (파이썬이 포트를 점유해야 하므로).

## 3. 파이썬 환경 준비
```
pip install pyserial
```
(tkinter는 파이썬 표준 라이브러리라 별도 설치 불필요)

## 4. 파이썬 UI 실행
```
python motor_ui.py
```
1. UI 상단 포트 콤보박스에서 아두이노가 연결된 COM 포트 선택 (안 보이면 "새로고침" 클릭).
2. "Connect" 버튼 클릭 → 연결되면 "연결됨 (COMx)"로 표시.
3. 전진(▲)/후진(▼)/좌(◀)/우(▶) 버튼을 누르고 있는 동안 해당 방향으로 이동(속도 조절 없이 항상 최대 속도), 손을 떼면 자동 정지.
   - 좌/우는 제자리 회전(피벗턴): 한쪽 모터는 정방향, 반대쪽은 역방향으로 동시 구동.
4. "정지(■)" 버튼으로 언제든 즉시 정지 가능.
5. 하단 로그창에서 아두이노가 보내는 ACK 응답 확인.

## 5. 종료
- UI 창을 닫으면 시리얼 연결이 자동으로 해제된다.

## 문제 해결
- 포트가 목록에 없음 : 아두이노 드라이버/케이블 확인 후 "새로고침" 클릭.
- 연결 실패 메시지 : 다른 프로그램(시리얼 모니터 등)이 같은 포트를 점유 중인지 확인.
- 모터가 반응 안 함 : 배선(IN1~IN4 핀 번호), ENA/ENB 고정 배선(HIGH) 상태, 모터 전원(18650) 연결 여부, GND 공통 연결 확인.
- 모터 방향이 반대로 동작 : 해당 모터의 IN1/IN2 배선을 서로 바꾸거나(하드웨어), `Motor.cpp`의 `setSpeed`에서 DIR1/DIR2 HIGH·LOW 조합을 반대로 수정.
- 좌/우 회전 방향이 반대로 느껴짐 : `motor_ui.py`의 `_turn_left` / `_turn_right` 함수에서 부호를 서로 바꿔 조정.
